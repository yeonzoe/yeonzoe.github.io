# Handoff 문서 생성

현재 작업 컨텍스트를 기반으로 다음 세션 인계용 문서를 생성한다.

## 사용법

```
/handoff [작업명]
```

---

## 실행 절차

### Step 1: 컨텍스트 수집

현재 대화에서 다음 정보를 추출한다:

1. **작업명**: `$ARGUMENTS` 또는 대화에서 추론
2. **변경 파일 목록**: `git status` 기준
3. **완료 내용**: 실제로 끝난 것
4. **미완료 내용**: 남은 TODO
5. **다음 세션 시작 방법**
6. **수동으로 해야 할 것** (GitHub Pages 설정 등)

### Step 2: 문서 생성

저장 위치: `.claude/handoffs/{YYYY-MM-DD}_{작업명}.md`

```markdown
# Handoff: {작업명}

날짜: {YYYY-MM-DD}

---

## TL;DR

> {한 줄 요약}

---

## 완료 (Done)
- [x] ...

## 남은 작업 (TODO)
- [ ] ...

---

## 다음 세션 시작 시

{첫 번째로 해야 할 것}

---

## 관련 파일

| 파일 | 역할 |
|------|------|
| index.html | 포트폴리오 메인 |

---

## 수동 작업 필요

| 항목 | 방법 |
|------|------|
| ... | ... |
```

---

## 주의사항

- 개인정보(전화번호 등)는 handoff에 포함하지 않는다
- 시크릿/토큰은 절대 포함하지 않는다
