Why these tools?
  • pytest — Python 기반으로 QA팀 학습 곡선 최소화, fixture/marker로 환경별 유연한 실행

  • Allure — 히스토리 트렌드, GitHub Pages 자동 배포, Slack 연동

Key Decisions
  • 왜 AI인가? — 4명 팀에서 에픽당 250+ TC 수동 작성 불가능. 토큰 비용 < 사람 시간 비용
  • 왜 정제된 PRD가 핵심인가? — 프롬프트 최적화보다 정확한 입력(정제된 PRD) 제공이 ROI 높음
  • 왜 3-Bucket인가? — UNCLEAR 전부를 "사람 확인"으로 미루면 병목. 리소스 투입 우선순위 결정용
  • 왜 23개 기준인가? — 오류 원인을 구조/커버리지/정확성/도메인/모호함으로 분류해야 개선 방향이 명확
